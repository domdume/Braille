# Rama Documentación - Proyecto Braille

Esta rama contiene toda la documentación completa del proyecto, organizada en carpetas específicas para facilitar su consulta y mantenimiento.

## Estructura de la Documentación

La documentación del proyecto está organizada en las siguientes carpetas:

### 1. Diseño Arquitectónico
**Carpeta:** `DiseñoArquitectonico/`

Contiene el diseño arquitectónico de alto nivel del sistema, incluyendo:
- Diagramas de arquitectura del sistema
- Descripción de componentes principales
- Patrones de diseño utilizados

### 2. Documentación del Ambiente de Desarrollo
**Carpeta:** `Documentacion Ambiente de Desarrollo/`

Incluye toda la información relacionada con el entorno de desarrollo:
- Herramientas seleccionadas para el desarrollo
- Flujo de trabajo del equipo
- Estrategia de ramificación (branching strategy)
- Configuración del entorno de desarrollo

### 3. Documentación Técnica
**Carpeta:** `Documentación Técnica/`

Documentación técnica detallada del código fuente:
- JavaDoc y comentarios del código
- Especificación de APIs

### 4. Documentación de Casos de Prueba
**Carpeta:** `Documentación casos de prueba/`

Contiene los casos de prueba y sus resultados:
- Especificación de casos de prueba
- Resultados de ejecución de cada caso
- Análisis de casos de prueba no exitosos
- Soluciones implementadas
- Re-ejecución y validación de casos corregidos

### 5. Manual de Instalación
**Carpeta:** `Manual Instalación/`

Guía paso a paso para la instalación del sistema:
- Requisitos previos
- Proceso de instalación
- Configuración inicial
- Resolución de problemas comunes

### 6. Manual de Usuario
**Carpeta:** `Manual de usuario/`

Documentación orientada al usuario final:
- Guía de uso del sistema
- Funcionalidades principales
- Ejemplos de uso
- Preguntas frecuentes (FAQ)

**Fecha de última actualización:** 2025-11-24
